# Agent Scoped MCP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add agent scoped MCP server management and runtime MCP tool access for Chat, Conn, and Team agent profiles.

**Architecture:** Store MCP server configs under each agent profile data directory, expose scoped CRUD/test routes, resolve enabled MCP servers into background snapshots, and inject one token-efficient `mcp` proxy tool into sessions. Use the official MCP TypeScript client package for stdio protocol handling and keep the first version per-call connected for reliable Windows cleanup.

**Tech Stack:** Fastify, TypeScript, Node test runner, TypeBox tool schemas, pi-coding-agent custom tools, official MCP TypeScript client, existing standalone Agent 管理台 UI.
