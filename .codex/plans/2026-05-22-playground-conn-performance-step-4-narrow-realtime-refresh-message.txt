本步目标：
- Coalesce And Narrow Realtime Refresh
- SSE/notification 事件不要每次都触发 `loadData()` 全量刷新。
- 后台任务事件默认只刷新 conn 相关数据，并做短窗口合并。

当前真实背景：
- `connectSSE()` 当前每次 `message` 都调用 `loadData()`。
- `loadData()` 会重拉 conns、agents、browsers、model config。
- `NotificationBroadcastEvent` 有 `source`、`sourceId`、`runId`、`kind`、`createdAt` 等字段，可用于识别 conn 事件。

- src/routes/notification-route-utils.ts
- src/agent/notification-hub.ts

- test/server.test.ts
- 相关 conn/notification tests

实现要求：
- 在 SSE message handler 中解析 `event.data`。
- `source === "conn"` 时走 conn-scoped refresh scheduler。
- 新增短窗口合并，例如 500ms；多个通知只触发一次实际刷新。
- notification 默认只刷新 `/v1/conns`。
- 不要在 notification 路径重拉 `/v1/agents`、`/v1/browsers`、`/v1/model-config`。
- 如果 selected conn 被影响且它的 run history 已加载，只刷新必要的 selected run first page 或 affected run detail。
- 避免并发重复刷新，后来的事件合并到 pending refresh。
- 手动刷新按钮仍能刷新用户需要的完整当前状态。

测试先行：
- 行为测试：多次 notification 在合并窗口内只触发一次 conn refresh。
- 测试 notification refresh 不调用 agent/browser/model loaders。
- 测试非 conn event 不会触发全量 `loadData()`。
- 测试 manual refresh 仍可刷新当前页面所需数据。
