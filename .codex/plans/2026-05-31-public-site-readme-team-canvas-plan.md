# UGK CLAW 官网：可信 Task 交付主线

## 背景

UGK CLAW 官网不能只介绍“Agent 画板”或“工作小组”这种界面形态。产品真正要解决的问题是：普通 LLM / Agent 即使模型能力足够，也会出现幻觉、漏项、偷工减料、伪造结果和任务遵从性不稳定。生产环境里，哪怕只有很小概率的不可信结果，也不能直接交付。

另一个核心痛点是复用。复杂任务可以在长对话里被调试出来并沉淀成 Skill，但长对话上下文会污染后续运行。Task 的价值在于把“干净会话 + 完整 Skill + 必要资料 + 期望产物”固定下来，让复用不依赖一段混乱的聊天历史。

## 核心定位

对外官网主叙事改为：

**UGK CLAW 是面向生产环境的 Agent 任务验收与编排工作台。**

首屏和关键段落围绕四个关键词组织：

- `Task`：干净会话承载可复用 Skill，减少上下文污染。
- `Worker`：执行固定 Task 或 Workflow。
- `Checker`：独立验收输出、证据、错误和遗漏。
- `Workflow`：把可信 Task 串联或并联，构成复杂任务。

## 信息架构

1. 首屏：让每个 Agent 任务都可验收。
   - 直接说明它不是普通聊天 UI，而是任务验收与编排工作台。
   - 主 CTA 指向“它解决什么问题”和“可信 Task 怎么产生”。
   - 不展示内部端口、localhost、开发命令。

2. 痛点段：普通 Agent 对话不适合直接进生产。
   - 讲清幻觉、遵从性、偷工减料、伪造结果。
   - 强调生产环境不能接受不可信交付。

3. 可信机制段：从污染上下文到可信 Workflow。
   - 显示链路：对话污染 / 幻觉风险 -> 干净 Task -> Worker 执行 -> Checker 验收 -> Workflow 编排。
   - 用短文案解释每一步的责任边界。

4. 能力段：从会聊天，变成可交付。
   - 隔离 Task。
   - 防止上下文污染。
   - Worker 执行。
   - Checker 验收。
   - 每个章节配独立视觉资产，数字和标题由 HTML/CSS 控制。

5. 工作小组段：工作小组不是包装，是验收结构。
   - Leader 组织任务。
   - Worker 执行任务。
   - Checker 拦截不合格输出。

6. 任务流转段：可信 Task 怎么产生。
   - 沉淀 Skill。
   - 放进干净 Task。
   - 执行并保留证据。
   - 审核后再编排。

7. 亮点段：可信交付。
   - 过程可观察。
   - 上下文可控。
   - 结果可验收。
   - Task 可编排。

## 视觉原则

- 成熟克制，不做廉价 AI 炫光堆叠。
- 概念视觉只承担品牌和场景表达，真实产品截图承担可信度。
- 数字章节、标题、说明全部由网页排版控制，生成图不承载文字。
- 对外页面只讲功能、价值和产品亮点；端口、启动命令、运行细节留在 README。

## 实现范围

- `src/ui/public-site.ts`
  - 官网首页 HTML/CSS。
  - 首屏、痛点、可信机制、能力章节、角色模型、任务流转、亮点和文档入口。
- `src/routes/public-site.ts`
  - `GET /` 首页。
  - `/site-assets/:fileName` 只允许图片资产，并防路径穿越。
- `docs/assets/*`
  - 概念主视觉、真实产品截图、角色素材、能力章节视觉素材。
- `test/server.test.ts`
  - 首页核心定位断言。
  - 图片资产可访问断言。
  - 禁止对外首页出现内部端口、localhost、开发端口等实现细节。

## 验收标准

- 官网首屏能在 10 秒内让新用户理解：UGK CLAW 解决 Agent 任务不可信的问题。
- 页面明确解释 Task、Skill、Worker、Checker、Workflow 的关系。
- 页面明确表达“干净 Task + Checker 验收”是可信交付的核心。
- 页面不伪造公开工作流 demo。
- 页面不暴露内部端口、localhost 或启动命令。
- 桌面和移动视口无横向溢出，按钮、标题、图片不互相遮挡。
- 图片资产全部从 `/site-assets` 安全提供。

## 测试计划

- 服务端测试：
  - `GET /` 返回官网 HTML。
  - 首页包含可信 Task、Checker 验收、干净会话、Workflow 编排等关键定位。
  - 首页不包含内部端口、localhost、开发端口和启动命令。
  - `/site-assets/*.png` 可访问。
  - 路径穿越返回 404。
  - `/playground` 不受首页路由影响。
- 静态检查：
  - `npx tsc --noEmit`
  - `npm run design:lint`
  - `git diff --check`
- 浏览器验收：
  - 桌面首屏标题、CTA、主视觉完整。
  - 桌面关键段落能读出痛点和解决机制。
  - 移动端无横向溢出。
  - 所有图片加载成功。
