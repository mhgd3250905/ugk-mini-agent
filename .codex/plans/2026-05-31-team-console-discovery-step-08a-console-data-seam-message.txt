实现要求：

1. 在 Team Console API abstraction 增加 `listGeneratedTasks(discoveryTaskId, options?)`。
   - Live API 调 `GET /v1/team/tasks/:taskId/generated-tasks`。
   - `taskId` 必须 URL encode。
   - `includeArchived: true` 时追加查询参数。
   - 支持 `{ tasks }` 响应；可以兼容 bare array。
   - `404` 返回 `[]`，不要让旧 backend 把 console 整体打挂。

2. Mock API/fixtures 增加 Discovery root + active/stale generated child fixtures。
   - `listTasks()` 仍只返回 root Tasks。
   - generated children 只能通过 `listGeneratedTasks()` 取到。
   - 默认排除 archived generated Tasks，`includeArchived` 才包含。

3. `useTeamConsoleLiveData()` 增加非视觉状态：
   - `generatedTasksByDiscoveryTaskId: Record<string, TeamCanvasTask[]>`
   - 推荐加 `discoverySummariesByTaskId`，至少包含 generated 总数、active 数、stale 数、running generated run 数。
   - 初始 live load / refresh 根据 root Tasks 中 `canvasKind === "discovery"` 的节点拉 generated catalog。
   - generated Tasks 不进入 root `tasks` state，不出现在主 canvas/root list。
   - 如果拉到了 generated children，把它们的 run summaries 放进现有 `taskRunsByTaskId`，给后续 UI 复用。
   - 没有 Discovery roots 时，不要调用 generated catalog endpoint。
   - generated catalog 拉取失败不能清空已经加载的 root Tasks；404 按空 catalog 处理。
   - failed dispatch count 先不要强行做，除非完全不用扩 observer/attempt 读取；这个可以留给后续 UI/observer 步。
